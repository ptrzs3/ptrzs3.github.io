关系型数据模型中的数据依赖与范式 Link to heading

在关系数据库中每个关系都需要进行规范化，使之达到一定的规范化程度，从而提高数据的结构性、共享性、一致性和可操作性。关系模型原理的核心内容就是规范化概念，规范化是把数据库组织在保持存储数据完整性的同时最小化冗余数据的结构的过程。规范化的数据库必须符合关系模型的范式规则。范式可以防止在使用数据库时出现数据不一致的情况，并防止数据丢失。关系模型有第一范式、第二范式、第三范式和BCNF范式等，高级范式从属于低级范式。

第一范式

  1. 数据组的每个属性只能包含一个值
  2. 关系中的每个数组必须包含相同数量的值
  3. 关系中的每个数组一定不能相通

第二范式：在满足第一范式要求的基础上，要求每一条数据必须可以被唯一区分，及非主属性完全依赖于主关键字

第三范式：在满足第二范式要求的基础上，要求数据库表中不存在已被其他表包含的非主关键字信息

在实际工作中，将数据解规范化是通用的惯例，也就是违反范式，尤其是第二、第三范式，以提高性能或减少复杂性

  • Connectors：与不同编程语言交互的接口
  • Connection Pool：管理缓冲用户连接、线程处理等需要缓存的需求
  • SQL INTerface：接受用户操作并返回结果

  • Engine：存储引擎。从mysql5.5之后，InnoDB是mysql的默认引擎

数据库表创建 Link to heading

  • 创建表

    CREATE TABLE <table_name>
(
<field> <type> [column CONSTRAINT] [default],
[table CONSTRAINT]
);

  • 使用表：use <table_name>;

  • 查看所有表名称：SHOW tables；

  • 主键约束

    一个表中可以有一个或多个主键，即单字段主键和联合主键

    /* 通过列级约束来指定主键 */
CREATE TABLE student
(
  /* 给id字段设置为从1开始自增*/
	id INT(11) PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(255) not null,
  /*给score字段添加默认约束*/
  score FLOAT default 0.0
);

    /* 也可以添加表级约束指定主键 */
CREATE TABLE student
(
	id INT(11) AUTO_INCREMENT,
  name VARCHAR(255) not null,
  score FLOAT,
  PRIMARY KEY(id)
);

    /* 通过表级约束指定联合主键 */
CREATE TABLE student
(
	id INT(11) PRIMARY KEY AUTO_INCREMENT,
  /* 给name字段添加非空约束*/
  name VARCHAR(255) not null,
  score FLOAT,
  PRIMARY KEY (id, name)
);

  • 外键约束

    外键用来在两个或多个表之间建立连接，一个表可以有多个外键，外键可以不是本表的主键，但一定是另一个表的主键。

    外键的值可以为空，如果不为空，则外键的值必须与另一个表中的某个主键值相等。

    创建外键的语法

    /* 将从表的主键注册为主表的外键，由从表维护两者关系 */
[CONSTRAINT <foreign_key_name>] FOREIGN KEY field_1,[field_2,...]
REFERENCES <master_table_name> primary_key_1[,primary_key_2,...]

    /* 创建主表 */
CREATE TABLE student
(
	id INT(10) PRIMARY KEY,
	/*给name字段添加唯一性约束*/
  /*可以将多个字段定义为UNIQUE，凡是定义为UNIQUE的列，值不能有重复*/
  name VARCHAR(20) UNIQUE,
  score FLOAT,
  school INT(3)
)
/* 创建从表 */
CREATE TABLE school
(
  id INT(3) PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(20),
  CONSTRAINT fk_stu_scl FOREIGN KEY (id) REFERENCES student(school)
  /* 将本表的id注册为外键，与表student字段school相关联*/
  /* school表作为从表，student表作为主表*/
);

数据库表修改 Link to heading

  • 修改表名

    ALTER TABLE <old_table_name> RENAME [to] <new_table_name>;

  • 修改字段类型

    ALTER TABLE <table_name> MODIFY <filed> <new_type>;

  • 修改字段名

    ALTER TABLE <table_name> change <old_field> <new_field> <new_type>;

  • 添加字段

    /* 添加新字段并指定字段位置：FIRST添加到首列，AFTER添加到指定字段后一列*/
ALTER TABLE <table_name> ADD <new_field> <type> [CONSTRAINT] [FIRST | AFTER <filed_name>];

  • 删除字段

    ALTER TABLE <table_name> DROP <field>;

  • 修改字段排序

    /*将某字段放在第一列或者指定字段后一列*/
ALTER TABLE <table_name> MODIFY <field_1> <type_of_field_1> <FIRST | AFTER <field_2>>;

  • 更改表存储引擎

    ALTER TABLE <table_name> ENGINE=<new_engine>;

  • 解除外键约束

    ALTER TABLE <table_name> DROP FOREIGN KEY <key_name>;

  • 删除表

    DROP TABLE <table_name>;
/*如果主表存在外键约束，需要先解除外键约束再删除主表，否则拒绝删除*/

mysql数据类型 Link to heading

mysql支持的数据类型主要有数值类型、日期/时间类型、字符串类型和二进制类型。

数值类型

  • 整数类型：TINYINT, SMALLINT, MEDIUMINT, INT, BIGINT

    声明方式：<type>[M]，M表示最大显示宽度，如果实际数据小于这个宽度，则左补空格，超过这个长度且不超取值范围仍然正常显示而不会截断

    Type Storage(Bytes) Comment
    TINYINT 1
    SMALLINT 2
    MEDIUMINT 3
    INT 4
    BIGINT 8

  • 浮点数和定点数：FLOAT, DOUBLE, DECIMAL

    声明方式：<type>[M, D]，M表示最大显示宽度，D表示小数显示宽度。对于DECIMAL，默认M=10，D=0；对于FLOAT和DOUBLE，默认保存为硬件支持的最大精度

    Type Storage(Bytes) Comment
    FLOAT 4 当数值超过设定的精度时四舍五入而不会警告
    DOUBLE 8 当数值超过设定的精度时四舍五入而不会警告
    DECIMAL M + 2 当数值超过设定的精度时四舍五入且发出警告；当表示货币、科学数据等对精度要求比较高的类型，尽量选择DECIMAL类型，以字符串形式保存

  • 日期和时间类型：YEAR, TIME, DATE, DATETIME, TIMESTAMP

    Type Storage Format Range Storage(Bytes) Comment
    YEAR YYYY 1901~2155 1 插入时可以是数值形式或者字符串形式
    TIME HH:MM:SS -835:59:59 ~ 835:59:59 3 插入时以字符串形式。如果不以冒号分隔，则自右向左解析，即认为最右两个字符为秒；以冒号分隔，则输入形式为’D HH:MM:SS’。最终的存储格式都是HH:MM:SS。非法数据被解析为00:00:00。可以使用CURRENT_TIME和NOW()获取当前时间
    DATE YYYY-MM-DD 1000-01-01 ~ 9999-12-31 3 插入时可以是字符串形式或者数值形式。以两位数表示年份，00～69被转换为2000～2069，70～99被转换为1970～1999。可以使用CURRENT_DATE和NOW()获取当前日期
    DATETIME YYYY-MM-DD HH:MM:SS 1000-01-01 00:00:00 ~ 9999-12-31 23:59:59 8 插入时可以是字符串形式或者数值形式，如'YYYY-MM-DD HH:MM:SS'或者YYMMDDHHMMSS
    TIMESTAMP YYYY-MM-DD HH:MM:SS 1970-01-01 00:00:01 ~ 2038-01-19 03:14:07 4 显示宽度默认为19位

  • 文本字符串类型：CHAR, VARCHAR, TINYTEXT, TEXT, MEDIUMTEXT, LONGTEXT, ENUM, SET. 文本字符串支持区分大小写和模式匹配

    Type Storage(Bytes) Comment
    CHAR(M: 1~255) M, 每个字符占用一个Bytes 定长文本字符串，长度不够右补空格
    VARCHAR(M: 1~255) L+1, L(<=M)为字符串实际长度，多出来的一个字节用来记录字符串长度 变长文本字符串
    TINYTEXT L+1, L<28
    TEXT L+2, L<216
    MEDIUMTEXT L+3, L<224
    LONGTEXT L+4, L<232
    ENUM 1或2 枚举类型的允许值在创建表时规定，最多支持65535个值。每个枚举字段只允许一个枚举值
    SET SET类型的允许值在创建表时规定，每个SET字段最多允许64个SET值 
    	CREATE TABLE <table_name> (<filed> ENUM('value_1', 'value_2'));
	/* 枚举类型字段如果允许为NULL，则该字段默认为NULL，否则默认为第一个枚举值 */
	/* 枚举值在mysql中存储的是索引编号，从1开始存储 */
	/* 枚举值NULL为自动添加，索引为NULL；''为自动添加，索引为0 */
	/* 插入数据时可以使用枚举值，也可以使用索引编号，非法值则报错，并且每个字段只允许一个枚举值或索引编号 */

    CREATE TABLE <table_name> (<field> SET('value_1', 'value_2'));
/* 重复值被自动剔除 */
/* 插入时可以是任意顺序，实际保存按定义顺序 */
/* 插入非法值会报错并禁止插入 */

  • 二进制字符串：BIT, BINARY, VARBINARY, TINYBLOB, BLOB

    Type Storage(Bytes) Comment
    BIT(M: 1~64) (M+7) / 8 位字段类型，M为每个数值占用的bit数，默认为1，长度不够左补0
    BINARY(M) M 定长二进制字符串，长度不够右补0
    VARBINARY(M) L+1 变长二进制字符串，用一个字节存储长度
    TINYBLOB(M) L+1, L<28 最大长度255字节
    BLOB(M) L+2, L<216 最大长度65535字节
    MEIDUMBLOB(M) L+3, L<224 最大长度16M字节
    LONGBLOB(M) L+4, L<232 最大长度4G字节

查询语句 Link to heading 

SELECT <field(s)> FROM <table_name | view_name> [WHERE conditions] [GROUP BY <field> [HAVING conditions]] [ORDER BY <field> [ASC | DESC]]

SELECT name FROM student WHERE age [NOT] IN (13, 14, 15);

SELECT name, score FROM student WHERE age BETWEEN 10 AND 15;

SELECT name FROM student WHERE name LIKE '_明%';
/* 李明， 张明达*/
/* 模糊查找，_匹配一个字符，%匹配任意长度字符*/

SELECT * FROM student WHERE courses IS [NOT] NULL;
/* 查询未[已]选课的学生信息 */

SELECT * FROM student
WHERE name LIKE '王%' OR name LIKE '张%'
ORDER BY age ASC, score DESC;

Fucntion Comment

SELECT name, AVG(score) FROM student GROUP BY name HAVING AVG(score) > 80;
/*
WHERE和HAVING的区别：
1. WHERE子句不能用统计函数，而HAVING可以
2. GROUP BY后面不可以用WHERE，而可以用HAVING
3. HAVING字句中可以使用字段别名，而WHERE不可以使用别名
4. HAVING筛选组，没有使用SELECT筛选的字段在HAVING中不能出现，而WHERE中可以出现
*/

SELECT <field> FROM <table_name> LIMIT [offset=0,]<length>;
SELECT * FROM student LIMIT 5;
SELECT * FROM student LIMIT 2,8;
/* 从第3行开始查询8行*/

连接查询 Link to heading

涉及两个及以上的表的查询称为连接查询。连接查询分为内连接、外连接和复合连接。连接查询要求参加连接的表必须有相同的字段，比如学生表和成绩表都有相同的学生学号字段。

内连接 Link to heading

连接查询中使用WHERE子句给出连接条件

WHERE [table_name.]<field_1> <operator> [table_name.]<field_2>
/*如果有必要则使用table_name.field来区分是哪个表的字段*/

SELECT stu.id, name, grade
FROM student stu, score sc # 给student表起别名stu，score表起别名sc。别名前后都要改
WHERE stu.id=sc.id;

# ANSI SQL，与上句等价
SELECT stu.id, name, grade
FROM student stu inner join score
on stu.id=score.id;

外连接 Link to heading

在内连接中，如果student表和score表中出现不相等的id，则对应的记录不会出现在查询结果中。

外连接分为左外连接和右外连接。对于左外连接，左表的所有记录和右表中满足条件的记录会出现在查询结果中，对于在左表中出现而右表中没有出现的，置为NULL；对于右外连接，右表的所有记录和左表中满足条件的记录会出现在查询结果中，对于在右表中出现而左表中没有出现的，置为NULL。

SELECT stu.id, name, grade
FROM student stu left join score sc
on stu.id=sc.id;
/* 查询student表中所有学生的ID、姓名和成绩，如果成绩表中没有的，置为NULL*/

SELECT stu.id, name, grade
FROM score sc right join student stu
on stu.id=sc.id;
/* 查询student表中所有学生的ID、姓名和成绩，如果成绩表中没有的，置为NULL*/

复合连接 Link to heading 

SELECT  id, grade
FROM student, score
WHERE student.id=score.id AND course='物理';
# WHERE子句中不仅可以跟连接条件，也可以跟限制条件

SELECT stu.id, name, course_name

WHERE stu.id=sc.id AND cr.id=sc.id;
# 两个表中没有共同的字段，通过第三张表进行连接
