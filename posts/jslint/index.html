ESLint Link to heading

在语法层面规范代码，兼顾排版效果

安装 Link to heading 

npm install eslint -D

初始化 Link to heading 

npm init @eslint/config

按照提示进行初始化后，eslint的规则文件在.eslintrc.yml中。

eslintrc文件将在ESLint v9.0.0被弃用，转而使用eslint.config.js

配置对象详解 Link to heading

配置文件（eslintrc）中包含多个对象，如extends、parserOptions、rules等。

extends Link to heading

entends对象可以继承多个规则，但是可以在rules对象中覆盖其中某些规则

默认继承eslint: recommended，如果在开发其他类型项目如Vue，可以添加继承plugin:vue/vue3-strongly-recommended等规则。

parserOptions Link to heading
rules Link to heading

rules对象包含用户具体配置的规则，如

rules:
  quotes:
    - warn
    - single
  no-unused-vars: off

eslint支持的所有规则：https://eslint.org/docs/latest/rules

执行 Link to heading 

npx eslint target.js

CLI：npx eslint [options] [file|dir|glob]*

eslint所有命令行参数：https://eslint.org/docs/latest/use/command-line-interface

痛点 Link to heading

若要查看是否有不规范代码，需要重新执行命令并从命令行中分析，或者npx eslint --ext .js --fix自动修复错误，而不能实时查看报错提醒。

解决方案 Link to heading

安装ESLint插件，实时提醒不规范代码

Prettier Link to heading

在排版层面规范代码，并不具有检查代码规范性的功能，是ESLint的补充

安装 Link to heading 

npm install --save-dev --save-exact prettier

初始化 Link to heading

创建一个空的.prettierrc文件，亦可创建.prettierignore告知prettier跳过某些不需要格式化的文件

node --eval "fs.writeFileSync('.prettierrc','{}\n')"

配置 Link to heading

prettier支持的所有规则：https://prettier.io/docs/en/options

printWidth: 80 # 每行代码最多80（推荐值）个字符
tabWidth: 4 # 每个缩进级占几个空格；如果这里没有设置，将读取.editorconfig的indent_size或者tab_width，否则覆盖之
useTabs: false # 使用tab键还是空格键控制缩进；如果这里没有设置，将读取.editorconfig的indent_style，否则覆盖之
semi: true # 在语句结尾是否使用分号
singleQuote: true # 使用单引号而不是双引号
quoteProps: as-needed # 需要在对象字面属性名称周围加上引号
jsxSingleQuote: false # 在JSX中使用单引号而不是双引号
trailingComma: all # 打印尾随逗号
bracketSpacing: true # 打印对象字面量间的空格
bracketSameLine: false # 将HTML标签的`>`放在行尾而不是新开一行
arrowParens: always # 在箭头函数上添加括号
# rangeStart: 0
# rangeEnd: Infinity # 只格式化文件的一部分，从rangeStart到rangeEnd
# parser: 在3.0.0已被移除，详见首行网页
# filePath: 指定用于推断使用哪个解析器的文件；
requirePragma: false # 只格式化那些包含pragma的文件
insertPragma: false # 格式化完成后在文件头插入pragma
proseWrap: preserve # 见官方解释
htmlWhitespaceSensitivity: css # 见官方解释
vueIndentScriptAndStyle: false # 是否缩进vue中<script>和<style>的代码
endOfLine: lf # 如果这里没有设置，将读取.editorconfig的end_of_line ，否则覆盖之
embeddedLanguageFormatting: "auto" # 控制 Prettier 是否格式化文件中嵌入的引号代码
singleAttributePerLine: false # 在 HTML、Vue 和 JSX 中每行强制只使用一个属性

执行 Link to heading 

npx prettier . --write

CLI：prettier [options] [file/dir/glob ...]

prettier的命令行：https://prettier.io/docs/en/cli

痛点 Link to heading

  1. 和ESLint一样，必须手动执行命令来规范代码
  2. 某些规则可能和ESLint冲突

解决方案 Link to heading

  1. 安装Prettier - Code formatter插件，实时查看不规范代码并修复

    设置为编辑器（VSCODE）的默认formatter（只针对JavaScript），并在保存时自动进行格式化：

    文件-首选项-设置（以json打开），添加以下内容

    {
"[javascript]": {
        "editor.defaultFormatter": "esbenp.prettier-vscode",
        "editor.formatOnSave": true
    }
}

  2. 安装eslint-config-prettier，并在ESLint配置文件中继承prettier规则

    If you use ESLint, install eslint-config-prettier to make ESLint and Prettier play nice with each other. It turns off all ESLint rules that are unnecessary or might conflict with Prettier.原文出处

    npm install --save-dev eslint-config-prettier

    Our recommended configuration automatically enables eslint-config-prettier to disable all formatting-related ESLint rules. 原文出处

    check if your configuration contains any rules that are unnecessary or conflict with Prettier. Here’s how to run it:

    npx eslint-config-prettier path/to/main.js

    确保prettier是ESLint继承的最后一个规则集，以确保prettier的规则不会被覆盖。但即使如此，prettier也可能会被rules中的规则覆盖（如果有重复规则）

eslint-plugin-prettier Link to heading

从ESLint运行prettier，并以ESLint的形式报错

安装 Link to heading 

npm install --save-dev eslint-plugin-prettier eslint-config-prettier
npm install --save-dev --save-exact prettier

eslint-plugin-prettier does not install Prettier or ESLint for you. You must install these yourself.

配置（eslintrc) Link to heading

继承plugin:prettier/recommended规则，一次性设置eslint-config-prettier和eslint-plugin-prettier

This will:

  • Enable the prettier/prettier rule.
  • Disable the arrow-body-style and prefer-arrow-callback rules which are problematic with this plugin - see the below for why.
  • Enable the eslint-config-prettier config which will turn off ESLint rules that conflict with Prettier.原文出处

prettier-eslint Link to heading

使用prettier和eslint修复JavaScript代码

Code ➡️ prettier ➡️ eslint –fix ➡️ Formatted Code ✨

要解决的问题 Link to heading

ESLint可以根据配置自动修复代码，而prettier是更强大的代码格式化工具。prettier是个有主见的工具，然后并不足够有主见，有时候与用户的意见相左。所以可能发生这样的情况：prettier将代码格式化后，ESLint就报错。

解决方案 Link to heading

本插件先对代码使用prettier进行格式化，然后将结果进行eslint --fix，这样就可以享受prettier和eslint两者的强大。

对于json、css、scss、less这些文件，只能使用prettier进行格式化，因为eslint不支持这些类型文件的lint

安装 Link to heading 

npm install --save-dev prettier-eslint
